Minikube Installation & Setup Guide (Ubuntu)
============================================

What is Minikube?
Minikube is a lightweight tool that lets you run a local Kubernetes cluster. It is mainly used for learning, development, and testing purposes.

System Requirements
===================
Before installing Minikube, ensure your system meets these requirements:
2 CPUs or more
2 GB RAM (minimum)
20 GB free disk space
Internet connection
Linux (x86-64 or ARM64)

Install Minikube (Latest Stable Release)
=======================================
Run the following commands to install Minikube on Ubuntu:

# Download latest Minikube binary
curl -LO https://github.com/kubernetes/minikube/releases/latest/download/minikube-linux-amd64

# Install it to /usr/local/bin
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# Clean up
rm minikube-linux-amd64


Minikube Drivers
================
Minikube needs a container or VM driver to run Kubernetes nodes.
In this guide, we’ll use Docker as the driver.

Install Docker
==============
Step 1: Set up Docker APT Repository
------------------------------------
# Update system
sudo apt-get update

# Install required packages
sudo apt-get install -y ca-certificates curl

# Add Docker’s GPG key
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add Docker repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Update repositories
sudo apt-get update

Step 2: Install Docker Engine
-----------------------------
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

Step 3: Verify Docker Installation
----------------------------------
sudo docker run hello-world

If you see the "Hello from Docker!" message, the installation is successful.

Step 4: Manage Docker as a Non-Root User
----------------------------------------
By default, Docker requires sudo. To run Docker without sudo:

# Create docker group
sudo groupadd docker

# Add your user to the docker group
sudo usermod -aG docker $USER

Log out and log back in (or restart your VM) for changes to take effect and varify using below command
docker run hello-world

Step 5: Enable Docker on System Boot
------------------------------------
To start Docker automatically after reboot:

sudo systemctl enable docker.service
sudo systemctl enable containerd.service


Configure Minikube to Use Docker Driver
=======================================
Make Docker the default driver for Minikube:
minikube config set driver docker

You’re now ready to start Minikube with Docker as the driver!
minikube start
